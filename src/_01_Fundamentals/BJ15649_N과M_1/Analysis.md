# 🔍 BJ15649: N과 M (1)

### 1. 문제 분석
- **유형**: 백트래킹 (Backtracking), DFS (깊이 우선 탐색)
- **목표**: 1부터 N까지 자연수 중에서 중복 없이 M개를 고른 수열을 모두 출력
- **제약 조건**: $1 \le M \le N \le 8$
- **시간 복잡도**: $O(_nP_m)$, 최대 $8! = 40,320$으로 시간 제한(1초) 내 여유롭게 통과 가능

### 2. 설계 및 로직 (State Space Tree)
본 문제는 모든 가능한 경우의 수를 탐색하되, **'중복 불가'**라는 조건에 따라 가지치기를 하는 백트래킹 구조이다.

- **State(상태)**: 현재 선택한 숫자의 개수 (`depth`)
- **Base Case (탈출 조건)**: `depth == M`이 되면 하나의 수열이 완성된 것이므로 출력 후 `return`
- **Recursive Step (탐색)**:
    1. 1부터 N까지 순회하며 아직 방문하지 않은 숫자(`!visited[i]`)를 찾음
    2. 해당 숫자를 방문 처리(`true`)하고 결과 배열에 저장
    3. 다음 깊이(`depth + 1`)로 재귀 호출
    4. **★중요★ 원상복구**: 재귀에서 돌아오면 해당 숫자의 방문 처리를 해제(`false`)하여 다른 경로에서 사용할 수 있도록 함



### 3. Java 실전 팁 & 고칠 점
- **출력 최적화**: 매번 `System.out.println`을 호출하는 대신 `StringBuilder`에 모아서 한 번에 출력하여 I/O 성능을 높임.
- **배열 인덱스 관리**: 출력용 배열(`array`)의 크기는 고정된 깊이 `M`으로 설정하고, 인덱스는 `depth` 변수를 활용해 순차적으로 채움.
- **백트래킹의 정석**: `visited[i] = true; -> dfs() -> visited[i] = false;`로 이어지는 흐름은 백트래킹의 가장 표준적인 패턴임.

### 4. 회고
- 처음 설계 시 `array[i-1] = i`와 같이 숫자에 인덱스를 고정하는 실수를 했으나, `depth`를 인덱스로 활용하여 순차적으로 값을 채우는 방식으로 교정함.
- `depth == M` 조건에서 `return`을 명시하여 불필요한 연산을 방지하는 것이 중요함을 배움.
- 이 구조를 익히면 이후 '중복 순열', '조합' 등 변형 문제도 조건문 하나로 쉽게 해결할 수 있을 것 같다.
