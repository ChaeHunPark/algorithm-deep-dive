# 🃏 알고리즘 분석: 블랙잭 (3-Sum 변형)

## 1. 문제 개요
- **목표**: $N$개의 카드 중 3장을 골라 합이 $M$을 넘지 않으면서 $M$에 가장 가까운 값을 찾기.
- **제약 조건**:
    - $3 \le N \le 100$
    - $10 \le M \le 300,000$
    - 시간 제한: 1.0초 (약 $10^8$번 연산 가능)

---

## 2. 복잡도 분석 (Complexity Analysis)

### A. 브루트 포스 방식 ($O(N^3)$)
- **방법**: 3중 `for`문을 사용하여 모든 카드 조합을 전수 조사.
- **계산**: $_NC_3 = \frac{N(N-1)(N-2)}{6}$
- **$N=100$일 때**: 약 161,700번 연산.
- **판단**: 1초 제한 내에서 매우 안전하게 통과 가능.

### B. 최적화 방식 ($O(N^2)$) - *선택한 방식*
- **방법**: 배열을 먼저 **정렬**한 뒤, 카드 하나를 고정하고 나머지 두 카드를 **투 포인터(Two Pointers)**로 탐색.
- **계산**: $O(N \log N)$ [정렬] + $O(N^2)$ [탐색]
- **$N=100$일 때**: 약 10,000번 연산.
- **판단**: $N$이 5,000 이상으로 커지는 대규모 데이터에서도 효율적으로 동작함.



---

## 3. 투 포인터 핵심 로직
1. 입력받은 배열을 오름차순으로 **정렬**한다.
2. 첫 번째 카드를 가리키는 인덱스 $i$를 $0$부터 $N-3$까지 순회한다 (고정).
3. 나머지 두 카드를 가리키는 포인터를 `left = i + 1`, `right = N - 1`로 설정한다.
4. `left < right`인 동안 다음을 반복한다:
    - `합 == M`: 정답을 찾았으므로 즉시 종료 및 $M$ 반환.
    - `합 < M`: 현재 합이 기존 최댓값보다 크면 갱신 후, 더 큰 합을 찾기 위해 `left++`.
    - `합 > M`: $M$을 초과했으므로 합을 줄이기 위해 `right--`.



---


## 4. 확장 가능성 (Scalability)
만약 $N$이 **1,000,000**까지 늘어난다면 $O(N^2)$으로도 해결이 불가능함. 이 경우 문제의 성격에 따라 비트마스킹, 다이내믹 프로그래밍(DP) 또는 수치적 제한을 활용한 다른 최적화 기법이 필요함.
