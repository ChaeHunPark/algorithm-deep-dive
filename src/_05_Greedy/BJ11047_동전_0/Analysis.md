# 📑 [Algorithm Analysis] 백준 11047 - 동전 0

### 1. 문제 핵심 요약
* **목표**: 주어진 $N$종류의 동전을 최소한으로 사용하여 합계 $K$원을 만들기.
* **핵심 조건**: $A_1 = 1$ 이며, $i \ge 2$ 인 경우 $A_i$ 는 $A_{i-1}$ 의 **배수**이다.

---

### 2. 논리적 추론 (Reasoning)
* **왜 Greedy인가?**
    * 일반적인 거스름돈 문제는 최적해를 위해 **DP(Dynamic Programming)**를 사용해야 함.
    * 하지만 이 문제는 **"큰 단위가 작은 단위의 배수"**라는 결정적인 조건이 있음.
    * 이는 큰 동전 1개를 쓰는 것이 작은 동전 여러 개를 쓰는 것보다 **항상 이득**임을 보장함.
    * 따라서 매 순간 가장 가치가 큰 동전을 선택하는 **탐욕적 선택(Greedy Choice)**이 전체 최적해를 보장함.
* **시간 복잡도**:
    * 동전의 종류 $N$만큼만 배열을 순회하면 되므로 **$O(N)$**.
    * $N$이 최대 10이므로 연산 속도가 매우 빠름.

---

### 3. 코드 설계 (Implementation)
1. **입력**: `BufferedReader`와 `StringTokenizer`를 사용하여 $N, K$ 및 동전 배열 `A[]`를 효율적으로 입력받음.
2. **역순 탐색**: 큰 동전부터 확인하기 위해 배열의 끝(`N-1`)부터 0까지 거꾸로 순회 (`for` 루프 활용).
3. **핵심 로직**:
    * `count += K / A[i]` : 현재 동전으로 만들 수 있는 최대 개수 합산.
    * `K %= A[i]` : 동전을 사용하고 남은 나머지 금액 갱신.

---

### 4. Self-Correction & 핵심 포인트
* **엣지 케이스 처리**: `if (moneyArr[i] <= K)` 조건을 통해 목표 금액과 동전 단위가 일치하는 경우도 누락 없이 처리함.
* **성능 최적화**: 단순히 동전을 하나씩 빼는 반복문 대신, **나눗셈(`/`)**과 **나머지(`%`)** 연산을 사용하여 연산 횟수를 최소화함.
* **수학적 해석**: 문제의 $A_i$ 표현을 배열 인덱스 `A[i]`로 정확히 치환하여 로직에 녹여냄.
