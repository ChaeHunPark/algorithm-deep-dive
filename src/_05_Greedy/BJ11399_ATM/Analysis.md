# 📑 [Algorithm Analysis] 백준 11399 - ATM

### 1. 문제 핵심 요약
* **목표**: $N$명의 사람이 돈을 인출하는 데 걸리는 총 시간(개인 대기 시간 + 인출 시간)의 합계를 최소화하기.
* **핵심 조건**: 앞사람이 인출하는 동안 뒷사람들은 그 시간을 고스란히 대기 시간으로 부담해야 함.

---

### 2. 논리적 추론 (Reasoning)
* **왜 Greedy인가?**
    * 각 사람의 인출 시간 $P_i$는 본인뿐만 아니라 뒤에 서 있는 모든 사람의 대기 시간에 포함됨.
    * 소요 시간이 긴 사람이 앞에 서면, 그 긴 시간이 뒤의 모든 사람에게 누적되어 전체 합이 기하급수적으로 커짐.
    * 따라서 매 순간 **"가장 짧은 인출 시간을 가진 사람"**을 선택하는 것이 전체 최적해를 보장함.
* **시간 복잡도**:
    * 입력 $O(N) +$ 정렬 $O(N \log N) +$ 누적합 계산 $O(N) +$ 최종 합산 $O(N)$.
    * 전체 복잡도는 가장 영향력이 큰 **$O(N \log N)$**에 수렴함.

---

### 3. 코드 설계 (Implementation)
1. **데이터 구조**: 정적 데이터 처리에 효율적이고 메모리 오버헤드가 적은 `int[]` 배열 사용.
2. **Sorting**: `Arrays.sort()`를 호출하여 $O(N \log N)$ 오름차순 정렬 수행.
3. **In-place 누적합**:
    - 별도의 배열 생성 없이 `peoples[i] += peoples[i-1]` 로직으로 기존 배열에 개인별 총 소요 시간을 갱신.
    - 갱신된 배열의 모든 요소를 `result` 변수에 합산하여 출력.

---

### 4. Self-Correction & 핵심 포인트
* **자료구조 선택**: 데이터가 한꺼번에 입력되는 고정형 문제이므로, 실시간 정렬이 필요한 `PriorityQueue` 대신 캐시 효율이 높은 **배열 정렬** 방식을 선택함.
* **공간 최적화**: 누적합 계산 시 새로운 공간을 할당하지 않고 입력 배열을 재사용($O(1)$ 추가 공간)하여 메모리 효율성을 극대화함.
* **자료형 고려**: $N=1,000$, $P_i=1,000$일 때 최대 합은 약 50만 수준으로 `int` 범위 내에서 안전하게 처리가 가능함을 확인함.
