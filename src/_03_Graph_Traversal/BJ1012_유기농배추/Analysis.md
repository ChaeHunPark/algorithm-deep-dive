# 🥬 1012번: 유기농 배추 (DFS) 분석

## 1. 문제 개요
* **핵심 목표**: 상하좌우로 인접한 배추들이 모여있는 구역(컴포넌트)의 총 개수를 구하기.
* **사용 알고리즘**: DFS (Depth First Search)
* **주요 특징**: 격자형(Grid) 그래프 탐색의 가장 전형적인 문제.

---

## 2. 알고리즘 설계 및 구현 (My Logic)

### 🏗️ 데이터 구조
1. **맵 정보 (`int[][] map`)**: 배추가 심어진 위치(1)와 빈 땅(0)을 저장하는 2차원 배열.
2. **방문 체크 (`boolean[][] visited`)**: 중복 카운팅을 방지하기 위한 방문 처리 배열.
3. **방향 벡터 (`dx`, `dy`)**: 상하좌우 인접 칸을 탐색하기 위한 오프셋 배열.

### ⚙️ 핵심 프로세스
1. **테스트 케이스 처리**: `while(T-- > 0)` 루프를 통해 매 케이스마다 맵과 방문 배열을 초기화.
2. **배추 찾기**: 전체 맵을 이중 for문으로 순회하며 '배추가 있고(1) 방문하지 않은(!visited)' 칸을 탐색.
3. **영역 탐색 (DFS)**:
    - 해당 지점에서 DFS를 시작하여 연결된 모든 배추를 방문 처리.
    - 한 번의 DFS가 끝나면 '지렁이 한 마리'(`count++`) 추가.

---

## 3. 핵심 인사이트 & 시행착오 (Trouble Shooting)

### ✅ 인덱스 오타 주의 (Critical)
* **실수**: `int ny = x + dy[i];`와 같이 좌표 변수를 혼동함.
* **교정**: `nx`에는 `x`를, `ny`에는 `y`를 더해주는 좌표 로직을 항상 더블 체크해야 함.
* **교훈**: 피로도가 높을 때 발생하는 전형적인 오타로, 변수명을 명확히 인지하며 코딩할 필요가 있음.

### ✅ 자바 배열의 기본값 활용
* **인사이트**: `new int[m][n]`으로 생성된 배열은 자동으로 `0`으로 초기화되므로, 매번 `Arrays.fill`을 돌릴 필요가 없음. 불필요한 연산을 줄여 성능을 최적화함.

### ✅ 시간 복잡도 분석
* **계산**: 한 테스트 케이스당 $O(N \times M)$.
* **성능**: 최대 $50 \times 50 = 2,500$번의 연산으로, 테스트 케이스가 많더라도 1초 제한 내에 매우 여유롭게 통과 가능함.

## 4. 회고
다익스트라와 같은 복잡한 알고리즘을 풀다가 가끔 이런 기초적인 그래프 탐색 문제를 풀면 로직의 흐름을 다시 한번 정교하게 다듬을 수 있다. 특히 좌표 탐색 시의 디테일한 실수(x/y 혼동)를 잡아내는 연습이 되었으며, 백엔드 개발자로서 자원(배열 초기화 등)의 효율적 사용에 대해 다시금 생각해보게 된 계기가 됨.
