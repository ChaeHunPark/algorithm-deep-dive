## 📝 Analysis.md (Step 03: Graph Traversal)

### 1. 문제 분석: BJ1260 (DFS와 BFS)
- **핵심 과제**: 주어진 그래프를 DFS(깊이 우선)와 BFS(너비 우선)로 탐색한 경로를 각각 출력.
- **주요 조건**:
  - 방문 가능한 정점이 여러 개일 경우 **번호가 작은 것**부터 방문.
  - 정점(N) ≤ 1,000 / 간선(M) ≤ 10,000.

### 2. 알고리즘 선택 및 기술적 근거
#### **A. 데이터 구조: 인접 리스트 (ArrayList<ArrayList<Integer>>)**
- **선택 이유**: 인접 행렬(O(N^2))은 N이 커질수록 빈 공간 낭비가 심함. 인접 리스트는 실제 연결된 간선만 저장하므로 메모리 효율이 높고 탐색 속도가 빠름.
- **사전 작업**: "작은 번호 우선" 조건을 만족시키기 위해 모든 리스트를 Collections.sort()로 오름차순 정렬.

#### **B. DFS (Depth-First Search)**
- **구현 방식**: **재귀(Recursion)** 호출.
- **방문 처리**: 함수가 호출되는 **시점(v)**에서 즉시 처리.
- **메커니즘**: 스택 메모리를 통해 한 경로를 끝까지 파고드는 특성을 구현.

#### **C. BFS (Breadth-First Search)**
- **구현 방식**: **Queue (ArrayDeque)** 활용.
- **방문 처리**: 큐에 **넣을 때(add)** 즉시 처리.
- **이유**: 큐에 중복된 노드가 쌓이는 것을 방지하여 메모리 낭비와 무한 루프를 차단함.

### 3. 시간 복잡도 계산
- **탐색 복잡도**: **O(N + M)**
  - 모든 정점(N)을 방문하고, 각 정점에 연결된 간선(M)을 한 번씩 확인.
- **정렬 복잡도**: O(M log M) (간선 정렬 비용)
- **최종**: 약 11,000번의 연산으로 해결 가능 (제한 시간 2초 대비 매우 여유로움).

### 4. 회고 및 백엔드 포인트
- **방문 처리 타이밍**: BFS에서 add 시점에 방문 처리를 하지 않을 경우 발생할 수 있는 큐 중복(Queue Explosion) 문제를 이해함.
- **효율적인 탐색**: 데이터의 밀도(Density)에 따라 리스트와 행렬 중 최적의 자료구조를 선택하는 능력을 배양함.
