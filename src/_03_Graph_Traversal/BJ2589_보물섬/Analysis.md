# [BOJ] 2178 - 미로 탐색 (Java)

## 1. 문제 분석
* **목표**: (0, 0) 지점에서 출발하여 (N-1, M-1) 지점까지 도달하는 **최소 칸 수**를 계산.
* **이동 조건**: 상하좌우로 인접한 칸만 이동 가능하며, 길(1)과 벽(0)이 명확히 구분됨.
* **특이사항**: 시작 지점과 도착 지점도 칸 수에 포함됨.

---

## 2. 기술적 접근: 왜 BFS인가?
* **최단 거리의 보장**: DFS는 한 경로를 끝까지 탐색하므로 처음 목적지에 도달한 경로가 최단이라는 보장이 없으나, **BFS는 시작점에서 가까운 곳부터 물결처럼 확장(Level-order Traversal)**하므로 처음 도착하는 순간이 무조건 최단 거리임.
* **거리 관리 전략**: 별도의 거리 배열을 만들거나, 기존 `map` 배열에 이전 칸의 값 + 1을 누적하여 시작점으로부터의 거리를 기록함.

---

## 3. 주요 로직 (Queue Mechanism)
1. **초기화**: `Queue<int[]>` 생성 후 시작 좌표 `(0, 0)` 삽입 및 `visited[0][0] = true` 처리.
2. **반복 탐색**: `while (!queue.isEmpty())` 루프를 돌며 `poll()`로 현재 좌표 추출.
3. **4방향 탐색**: 상하좌우(dx, dy)를 순회하며 다음 조건 검사:
    - **경계선 체크**: `nx, ny`가 배열 범위 내에 있는가?
    - **이동 가능 여부**: `map[nx][ny] == 1` (길)인가?
    - **방문 여부**: `!visited[nx][ny]` (미방문)인가?
4. **거리 갱신**: 조건 만족 시 `map[nx][ny] = map[currentX][currentY] + 1`로 값을 갱신하고 큐에 삽입.
5. **종료**: 큐에서 꺼낸 좌표가 `(N-1, M-1)`이 되는 순간 해당 값을 출력하고 종료.

---

## 4. 복잡도 분석
* **시간 복잡도**: $O(V + E)$
    * $V$ (정점): $N \times M$
    * $E$ (간선): 각 정점당 최대 4방향 이동
    * 최대 $100 \times 100 = 10,000$개의 칸을 방문하므로 제한 시간 내 매우 여유롭게 통과 가능.
* **공간 복잡도**: $O(N \times M)$
    * 지도 데이터 및 방문 처리를 위한 2차원 배열 사용.

---

## 5. 회고
* **BFS의 정석**: '가중치가 없는 그래프의 최단 거리'는 BFS라는 공식을 다시 한번 체득함.
* **배열 값의 활용**: 단순히 방문 여부만 체크하는 것이 아니라, `map` 배열 자체에 거리를 누적함으로써 메모리를 효율적으로 사용하고 직관적인 풀이가 가능했음.
* **큐 성능**: Java의 `LinkedList`보다 `ArrayDeque`를 사용하는 습관을 통해 미세한 성능 최적화를 고려함.
