## 📝 Analysis.md (Step 04: BFS & Shortest Path)

### 1. 문제 분석: BJ2178 (미로 탐색)
* **목표**: (0, 0)에서 출발하여 (N-1, M-1)까지 도달하는 **최소 칸 수** 계산.
* **특이사항**: 이동할 수 있는 칸(1)과 없는 칸(0)이 구분되어 있으며, 상하좌우로만 이동 가능.

### 2. 기술적 접근: 왜 BFS인가?
* **DFS vs BFS**:
    * **DFS**: 한 경로를 끝까지 파고들기 때문에 처음 도착한 경로가 최단 거리임을 보장하지 않음.
    * **BFS**: 시작점에서 거리가 1인 칸, 2인 칸 순서로 **물결처럼 확장**하며 탐색함.
* **최단 거리 보장**: BFS는 너비 우선으로 탐색하므로, **목적지에 처음 닿는 순간이 무조건 최단 거리**임이 수학적으로 보장됨.
* **거리 관리**: `visited` 배열로 중복 방문을 방지하고, `map[nx][ny] = map[x][y] + 1`을 통해 시작점으로부터의 거리를 칸마다 직접 기록함.



### 3. 주요 로직 (Queue Mechanism)
1. **초기화**: `Queue<int[]>` 생성, 시작 좌표 (0, 0) 삽입 및 `visited[0][0] = true`.
2. **반복 탐색**: `while (!queue.isEmpty())` 루프 실행.
    * `poll()`을 통해 현재 좌표 `(x, y)`를 꺼냄.
    * 현재 좌표가 목적지 `(N-1, M-1)`라면 즉시 `map[x][y]` 반환.
3. **방향 탐색**: 상하좌우 4방향 탐색 시 다음 조건 체크:
    * 경계선 내부인가? (`nx >= 0 && nx < n && ny >= 0 && ny < m`)
    * 길(1)이며 미방문 상태인가? (`map[nx][ny] == 1 && !visited[nx][ny]`)
4. **갱신 및 삽입**: 조건 만족 시 `map[nx][ny] = map[x][y] + 1`로 거리 갱신 후 큐에 삽입.

### 4. 시간 복잡도 분석
* **연산량**: 모든 칸(V)을 최대 한 번 방문하고, 각 칸에서 4방향(E)을 검사함.
* **최종**: $O(V + E) = O(N \times M)$.
* $100 \times 100$ 크기 기준 최대 1만 번 수준의 연산으로 제한 시간 내 여유롭게 통과 가능.

### 5. 회고
* **실수 교정**: BFS는 재귀가 아닌 **Queue**를 사용해야 함을 명확히 인지함.
* **데이터 업데이트**: 큐에서 꺼낸 객체(`poll`)의 데이터를 기반으로 다음 좌표를 계산해야 무한 루프를 방지할 수 있음.
