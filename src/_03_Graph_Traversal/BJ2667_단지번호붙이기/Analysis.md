## 📝 Analysis.md (Step 03-2: Connected Components)

### 1. 문제 분석: BJ2667 (단지번호붙이기)
- **핵심 과제**: 2차원 격자 지도에서 연결된 '1'(집)들의 묶음(단지)을 찾고, 단지 수와 각 단지의 크기를 계산.
- **연결 기준**: 상, 하, 좌, 우 (대각선 제외).

### 2. 기술적 접근
- **탐색 알고리즘**: **DFS (재귀)** 채택.
- **방문 처리 로직**:
    - `main`의 이중 for문이 전체 지도를 스캔하며 방문하지 않은 '1'을 발견하면 새로운 단지 탐색 시작.
    - DFS 내부에서 방문한 집은 `visited[x][y] = true`로 마킹하여 중복 탐색 방지.
- **좌표 이동**: `dx`, `dy` 배열을 활용한 4방향 탐색 및 경계선 체크(`nx, ny` 범위 확인).

### 3. 시간 복잡도 분석
- **지도 전체 스캔**: $O(N^2)$ (여기서 $N$은 지도의 한 변의 길이).
- **DFS 탐색**: 모든 칸을 최대 한 번씩 방문하므로 $O(N^2)$.
- **최종**: $O(N^2)$, $N=25$이므로 최대 625번의 연산으로 매우 효율적임.

### 4. 회고
- **재귀 반환값 활용**: DFS 함수가 자기 자신을 호출하며 결과값(집의 개수)을 합산하여 반환하는 구조를 통해 복잡한 전역 변수 없이 단지 크기를 계산함.
- **오타 주의**: `nx`, `ny` 계산 시 상하좌우 배열 인덱스를 혼동하지 않도록 주의가 필요함을 체감함.
