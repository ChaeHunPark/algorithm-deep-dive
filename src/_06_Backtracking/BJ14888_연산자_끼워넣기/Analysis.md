# 📑 [Algorithm Analysis] 백준 14888 - 연산자 끼워넣기

### 1. 문제 핵심 요약
* **목표**: $N$개의 수와 $N-1$개의 연산자가 주어졌을 때, 만들 수 있는 식의 결과가 최대인 것과 최소인 것을 구하기.
* **핵심 조건**: 연산자 우선순위를 무시하고 앞에서부터 진행하며, 나눗셈은 정수 나눗셈의 몫만 취함 (음수 나눗셈 조건 포함).

---

### 2. 논리적 추론 (Reasoning)
* **왜 백트래킹인가?**
    * 연산자를 배치하는 모든 가능한 순열(경우의 수)을 탐색해야 함.
    * $N$이 최대 11로 작기 때문에, 모든 경로를 탐색하되 '남은 연산자 개수'를 상태로 관리하며 가지치기하는 **DFS 기반 백트래킹**이 최적임.
* **상태 전이 설계**:
    * 재귀 함수의 인자로 `(현재까지의 누적 계산 결과, 다음 숫자의 인덱스)`를 넘겨주어 별도의 복잡한 자료구조 없이 상태를 유지함.
* **시간 복잡도**:
    * 최대 연산자 개수가 10개이므로, 중복 순열의 관점에서 최악의 경우에도 충분히 제한 시간 내 연산 가능.

---

### 3. 코드 설계 (Implementation)
1. **데이터 구조**: 연산자별 남은 개수를 `int[] operators = new int[4]` 배열에 담아 관리.
2. **재귀 함수 (`dfs`)**:
    - **기저 조건**: `idx == n` (모든 숫자를 다 사용했을 때) 최댓값/최솟값 갱신 후 `return`.
    - **유망성 검사**: `if (operators[i] > 0)`를 통해 남은 연산자가 있을 때만 탐색 진행.
3. **백트래킹 핵심 (상태 복구)**:
    - `operators[i]--` (연산자 사용) -> `dfs()` 호출 -> `operators[i]++` (연산자 반납).
    - 이 과정을 통해 하나의 경로 탐색이 끝나면 다음 경로를 위해 상태를 원복함.

---

### 4. Self-Correction & 핵심 포인트
* **초기값 설정**: `maxValue`는 `Integer.MIN_VALUE`, `minValue`는 `Integer.MAX_VALUE`로 설정하여 첫 갱신이 확실히 일어나도록 처리함.
* **나눗셈 처리**: 자바의 `/` 연산자가 문제에서 요구하는 "음수를 양수로 나눌 때의 처리 방식"과 동일하게 작동함을 확인하여 별도의 조건문 없이 간결하게 구현함.
* **매개변수 활용**: 누적 결과값을 재귀 인자로 넘김으로써 백트래킹 시 별도의 역연산 로직이 필요 없게 설계함.
