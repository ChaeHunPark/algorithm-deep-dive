# 🧠 DP(Dynamic Programming) 삼총사 학습 분석

오늘 학습한 세 문제는 모두 "최적의 경로(최솟값)를 찾는 DP"라는 공통점을 가지고 있습니다.
백엔드 개발자의 관점에서 각 문제의 핵심 로직과 차이점을 분석합니다.

---

### 1. [BJ1463] 1로 만들기 (나눗셈/뺄셈 혼합)
* **핵심 전략**: 숫자를 1로 줄여나가는 모든 경우의 수를 수첩(DP)에 기록.
* **로직 포인트**:
    - 일단 `dp[i] = dp[i-1] + 1`로 기본값(Default)을 세팅한다.
    - 2나 3으로 나누어질 때만 `Math.min`을 통해 더 빠른 길을 찾는다.
* **한줄평**: "DP의 가장 기초인 '과거 기록 컨닝'을 배우는 입문서."

### 2. [BJ2839] 설탕 배달 (조합/그리디 기반 DP)
* **핵심 전략**: 3kg와 5kg 봉투를 조합하여 정확한 무게를 맞춤.
* **로직 포인트**:
    - `i-3`번 칸과 `i-5`번 칸을 확인하여 배달 가능한 길인지 체크한다.
    - 만들 수 없는 무게는 `INF`로 처리하여 '막힌 길'임을 명시한다.
* **한줄평**: "모든 경로가 정답이 아닐 수 있음을 배우는 예외 처리의 정석."

### 3. [BJ14916] 거스름돈 (동전 최적화)
* **핵심 전략**: 2원과 5원 동전을 최소 개수로 조합.
* **로직 포인트**:
    - 설탕 배달과 구조가 99% 동일함.
    - 17원 사례처럼 "5원을 많이 쓰는 게 항상 유리한가?"를 `Math.min`이 판별하게 함.
* **한줄평**: "대입(=) 실수 하나로 로직이 깨질 수 있음을 배운 실전 최적화."

---

### 📊 DP 공통 아키텍처 요약
1. **정의**: `dp[i]`는 `i`를 만들기 위한 최소 횟수/개수.
2. **초기화**: 아주 작은 단위(`dp[2]`, `dp[3]`)의 정답을 먼저 적어둠.
3. **점화식**: `dp[i] = dp[i - 단위] + 1` (과거의 데이터 + 나의 시도 1번).
4. **갱신**: 여러 선택지 중 가장 작은 값을 `dp[i]`에 대입(`=`).
