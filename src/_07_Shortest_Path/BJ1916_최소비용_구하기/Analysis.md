# 🚌 1916번: 최소비용 구하기 (Dijkstra) 분석

## 1. 문제 개요
* **핵심 목표**: N개의 도시와 M개의 버스 노선이 주어질 때, 특정 도시 A에서 특정 도시 B로 가는 최소 비용을 계산.
* **사용 알고리즘**: 다익스트라 (Dijkstra) 알고리즘.
* **기술적 근거**: 가중치가 양수이며, 단일 출발점에서 특정 도착점까지의 최단 거리를 구하는 문제이므로 $O(E \log V)$의 복잡도를 가지는 우선순위 큐 기반 다익스트라가 최적임.

---

## 2. 알고리즘 설계 및 구현 (My Logic)

### 🏗️ 데이터 구조
1. **인접 리스트 (`ArrayList<Node>[] graph`)**: 각 도시 간의 연결 상태와 버스 비용을 저장.
2. **최단 거리 배열 (`int[] dist`)**: 시작점으로부터 각 도시까지의 현재 발견된 최소 비용을 기록. (초기값: `INF`)
3. **우선순위 큐 (`PriorityQueue<Node>`)**: 현재 가장 비용이 적은 노드를 먼저 선택하여 탐색 성능 최적화.

### ⚙️ 핵심 프로세스
1. **그래프 구축**: 버스 정보를 입력받아 인접 리스트에 저장.
2. **다익스트라 실행**:
    - 시작점의 거리를 `0`으로 설정하고 `PQ`에 삽입.
    - `PQ`에서 꺼낸 노드의 현재 비용(`curWeight`)이 이미 기록된 거리(`dist[curIdx]`)보다 크다면 무시 (이미 최적화된 경로가 존재함).
    - 인접 노드를 순회하며 `dist[curIdx] + next.weight`가 기존 거리보다 작을 경우 갱신 후 `PQ`에 삽입.
3. **결과 출력**: 목적지 도시(`end`)의 `dist` 값을 출력.

---

## 3. 핵심 인사이트 & 시행착오 (Trouble Shooting)

### ✅ 우선순위 큐와 객체 정렬
* `Node` 클래스에 `Comparable` 인터페이스를 구현하여 가중치(`weight`) 기준 오름차순 정렬이 되도록 설정함. 이는 다익스트라가 '그리디'하게 가장 짧은 경로를 먼저 탐색하게 하여 연산 횟수를 획기적으로 줄여줌.

### ✅ 중복 경로 처리
* 문제 조건에서 두 도시 사이에 여러 버스 노선이 있을 수 있다고 명시됨. 다익스트라 알고리즘 특성상 인접 리스트에 모든 간선을 추가하더라도 `if(dist[next.target] > ...)` 조건문에서 더 비싼 경로는 자연스럽게 걸러지므로 별도의 중복 제거 로직 없이 효율적으로 처리 가능함.

### ✅ 시간 복잡도 계산
* 도시 수 $N \le 1,000$, 버스 수 $M \le 100,000$.
* 복잡도는 $O(M \log N)$ 이며, 약 $100,000 \times 10 = 1,000,000$ 정도의 연산이 발생함.
* 자바의 1초 제한(약 1억 번 연산) 내에 매우 널널하게 통과 가능한 수치임을 확인함.

## 4. 회고
